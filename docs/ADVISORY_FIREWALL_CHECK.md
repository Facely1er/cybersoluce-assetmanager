# Advisory Firewall Check

## Purpose
Verify that ERMITS Advisory pages maintain proper boundaries and do not overclaim CyberSoluce outputs.

## Verification Checklist

### 1. Language Verification

#### Prohibited References
Verify ERMITS Advisory pages do NOT reference:
- [ ] "CyberSoluce findings" - Absent
- [ ] "Platform results" - Absent
- [ ] "Tool assessment outcomes" - Absent
- [ ] "Asset intelligence conclusions" - Absent
- [ ] Any authoritative claims from CyberSoluce - Absent

#### Allowed References
Verify ERMITS Advisory pages use:
- [ ] "Exploration" - Present
- [ ] "Clarification" - Present
- [ ] "Decision framing" - Present
- [ ] "Contextual input" - Present
- [ ] "Focus areas" - Present

### 2. Disclaimer Verification

Verify the following disclaimer is present (or equivalent):
- [ ] "Digital diagnostics inform discussions but do not replace advisory judgement"
- [ ] Or equivalent language protecting advisory credibility

### 3. Boundary Protection

#### Advisory Independence
- [ ] Advisory maintains independent judgment
- [ ] No reliance on CyberSoluce as authoritative source
- [ ] Advisory uses CyberSoluce as context only
- [ ] Advisory conclusions are independent

#### Credibility Protection
- [ ] Advisory credibility is fully protected
- [ ] No implication that advisory relies on tools
- [ ] Advisory judgment is clearly distinguished
- [ ] Expert analysis is emphasized

### 4. Stop Condition

**Question**: Is Advisory credibility fully protected?

- [ ] YES - Advisory maintains independent judgment
- [ ] YES - No overclaiming of CyberSoluce outputs
- [ ] YES - Disclaimer is present
- [ ] YES - Language is appropriate

## Expected Results

### ✅ Pass Criteria
- No prohibited references present
- Allowed language patterns used
- Disclaimer is present
- Advisory maintains independence
- Credibility is protected

### ❌ Fail Criteria
- Prohibited references found
- Advisory overclaims CyberSoluce outputs
- No disclaimer present
- Advisory appears dependent on tools
- Credibility is compromised

## Test Procedure

1. Navigate to ERMITS Advisory pages
2. Search for prohibited language patterns
3. Verify allowed language is used
4. Check for disclaimer presence
5. Verify advisory independence
6. Document any issues found
